Practica de JavaScript

JavaScript es un lenguaje de programación que se ejecuta tanto en el Frontend como en el Backend

JavaScript se usa en varios Frameworks:

  1. React
  2. Vue.js
  3. Angular
  4. Svelte/SvelteKit
  5. Next.js
  6. Nuxt.js
  7. Astro

Para programar en JavaScript usted hara uso de sus conocimientos en programación (básicos/intermedios)
Variable: locales, globales, estaticas/constantes
Estructuras de control (if/else)
Bucles (for/while/...)
Funciones

Tipos de Variables

  • let. locales - se usan dentro de una funcion o bloque de programacion
  • var: globales . las podemos usar en cualquier parte del desarrollo
  • const: constantes - variables con valores fijos, que no pueden cambiar su valor

Tipos de Datos

  • String: caracteres alfanumericos
  • Number: valores numericos
  • Booleano: true/false
  • Arrays: Listas["Elemento Liste #", "Elemento Lista #2", "Elemento Lista #N"]
  • Object - Diccionarios: NombreDiccionario{"clave1":"valor1", "clave2":"valor2", "claveN":"valorN"}
  • Null . Valor Indefnido

Programacion en JavaScript

  • Condiciones: if/else
  • Estrcuturas: switch
  • Ciclos/Bucles: for/while/do while
  • for...for - para recorrer colecciones (arrays)
  • for...in - para recorrer object/dicc (clave)

Momento-Luga rcuando se ejecuta JavaScript

  1. Script dentro del HTML - Script embebido en medio del HTML - (inline Script Early)
  2. Script dentro de un control - uso de los atributos en elementos HTML - (inline Events Handlers)
  3. Script al final (Antes de cerrar body) - Script embebido antes de cerrar el body - (inline Script Late)
  4. Archivo Externo (.js) - Script externo vinculado al HTML - (External Script)

Eventos Programables JavaScript dentro de una Página Web

Eventos para JavaScript para páginas webs
Tarea final de JavaScript: Desarrolle un ejemplo para cada evento enlistado a continuación.
  1. DOMContentLoaded

    2. El evento DOMContentLoaded se dispara cuando el navegador ha terminado de analizar completamente el documento HTML y construido el árbol del DOM, pero sin esperar a que se carguen imágenes, hojas de estilo u otros recursos externos. Es ideal para ejecutar scripts que necesiten manipular elementos del DOM de forma inmediata, garantizando que la estructura básica del documento ya esté lista para interactuar con JavaScript.

    El DOM aún no ha sido cargado.

  2. load

    3. El evento load ocurre cuando toda la página, incluidos los recursos como imágenes, hojas de estilo y scripts externos, ha sido completamente cargada. Es útil para ejecutar funciones que dependen de que todos los elementos visuales estén disponibles, como iniciar animaciones, medir elementos o cargar datos que requieren una interfaz totalmente renderizada.

  3. unload

    4. El evento unload se activa cuando el usuario está saliendo o recargando la página. Su función principal es permitir la limpieza de recursos, el cierre de conexiones o el almacenamiento de información antes de abandonar el sitio. Sin embargo, su uso se ha limitado en navegadores modernos por razones de seguridad y rendimiento, siendo reemplazado en muchos casos por beforeunload.

  4. beforeload

    5. El evento beforeunload se ejecuta justo antes de que la página comience a descargarse o cerrarse. Permite mostrar un cuadro de diálogo de confirmación para advertir al usuario sobre la pérdida de datos no guardados o interrumpir procesos en curso. Aunque su personalización visual está restringida por los navegadores, sigue siendo muy útil para proteger formularios o sesiones activas.

  5. visibilitychange

    6. El evento visibilitychange detecta cuando cambia la visibilidad del documento, por ejemplo, cuando el usuario cambia de pestaña, minimiza la ventana o regresa a la página. Se utiliza para pausar tareas en segundo plano, como animaciones o reproducción de video, y reanudarlas cuando la pestaña vuelve a estar visible, mejorando así el rendimiento y la experiencia del usuario.

  6. readystatechange

    7. El evento readystatechange se dispara cada vez que cambia el estado de carga del documento, reflejado en la propiedad document.readyState, que puede ser "loading", "interactive" o "complete". Es útil para controlar diferentes fases de carga, ejecutar código tan pronto el DOM está listo o esperar hasta que todos los recursos estén completamente cargados.

  7. fullscreenchange y fullscreenerror

    8. El evento fullscreenchange se activa cuando un elemento entra o sale del modo de pantalla completa, permitiendo adaptar la interfaz o mostrar mensajes contextuales. Por su parte, fullscreenerror se dispara cuando ocurre un error al intentar cambiar a dicho modo (por permisos o incompatibilidad). Ambos son esenciales para gestionar correctamente las experiencias multimedia o de visualización ampliada.

  8. copy, cut, paste

    9. Los eventos copy, cut y paste se ejecutan cuando el usuario copia, corta o pega contenido en la página. Permiten interceptar y personalizar las acciones del portapapeles, por ejemplo, para validar datos, modificar el texto antes de copiarlo o impedir que se pegue contenido no permitido. Son ampliamente usados en aplicaciones de texto, formularios y editores web.

    No se ha realizado ninguna acción

  9. selectionchange

    10. El evento selectionchange ocurre cada vez que cambia la selección de texto dentro del documento. Es útil para detectar qué parte del texto ha sido seleccionada por el usuario y reaccionar dinámicamente, como mostrar opciones contextuales, subrayar fragmentos o realizar búsquedas. Este evento es clave para implementar funciones interactivas en editores o visores de texto.

    Seleccionado algo

  10. scroll

    11. El evento scroll se dispara cuando el usuario desplaza la página o un elemento con barras de desplazamiento. Se utiliza comúnmente para efectos visuales como animaciones al hacer scroll, carga perezosa (lazy loading) de contenido o actualización de elementos en función de la posición actual. Aunque es muy potente, debe usarse con precaución para evitar afectar el rendimiento si se ejecuta con demasiada frecuencia.
